────────────────────────────────────
1. High-Level Explanation (Frontend Interview Level)
────────────────────────────────────
- Explain the concept as if answering a **senior frontend system design interview**.

  - What the concept/system is
  - Why it exists
  - When and where it is used
  - Its role in large-scale frontend applications

2. Deep-Dive Explanation (Senior / Staff Level)
────────────────────────────────────

  - Browser internals (rendering, event loop, memory, networking) where relevant
  - Data flow, state flow, and lifecycle
  - Performance implications (re-renders, bundle size, network, caching)
  - Scalability considerations (millions of users, CDN, micro-frontends)
  - Trade-offs (UX vs performance, complexity vs flexibility)

────────────────────────────────────
3. Clear Real-World Examples
────────────────────────────────────
- Provide concrete examples from:
  - FAANG-scale web applications
  - Dashboards, feeds, chat systems, e-commerce UIs, real-time systems

────────────────────────────────────
4. Interview-Oriented Explanation
────────────────────────────────────
- Provide:
  - A **crisp, confident sample interview answer** for 7+ years experience
  - Likely **follow-up questions** asked by frontend interviewers

────────────────────────────────────
5. Code Examples (When Applicable)
────────────────────────────────────
- Provide clean, minimal, interview-appropriate code snippets
  (React / framework-agnostic where possible).
- Explain:
  - Why the code is structured this way
  - How it affects performance, re-renders, and scalability
  - What you would change in production vs interview settings
- Avoid unnecessary boilerplate or tutorial-style code.

────────────────────────────────────
6. Why & How Summary
────────────────────────────────────
- **Why it matters** (user experience, performance, business impact)
- **How it works** (technical summary in simple but precise terms)
